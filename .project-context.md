## 1. High-Level Architecture

**Pattern**: Modular Monolith + Domain-Driven Design (DDD)  
**Language**: Go 1.25 (standard library only — no web frameworks)  
**Module name** (go.mod): `sentinel-flow`  
**Repository**: `github.com/arifrhm/sentinel-flow-event-driven-modular-monolith-ddd`

### Runtime Modes (toggled via env vars — no code change needed)

| Env Var | `memory` (default) | `postgres` / `redis` |
|---|---|---|
| `DATABASE_TYPE` | Thread-safe in-memory maps | PostgreSQL via `lib/pq` driver |
| `BROKER_TYPE` | Go buffered channels (fan-out) | Redis Pub/Sub via `go-redis/v9` |
| `LOG_FORMAT` | `text` (human-readable) | `json` (structured for log aggregators) |

## 2. Event Flow — End to End

```

  │
  ▼  POST /v1/events  (:8081)
┌─────────────────────────────────────────┐
│  Ingest Domain                          │
│  1. Validate required fields            │
│  2. CheckGDPRConsent(eventType, bool)   │
│  3. AnonymizeIP(ipStr) — mask last octet│
│  4. broker.PublishRaw(ctx, event)       │

  │ channel: sentinel_flow_raw

┌─────────────────────────────────────────┐
│  Fraud Domain (goroutine consumer)      │
```

## 3. Domain-by-Domain Reference

### 3.1 `pkg/domain/ingest`

| File | Responsibility |
|---|---|
| `consent.go` | `CheckGDPRConsent(eventType, bool)` — gates `click` and `view_item` events behind consent; `AnonymizeIP(ipStr)` — zeros last octet of IPv4, last 8 bytes of IPv6 |
| `handler.go` | `HandleIngest` — POST `/v1/events`; validates EventID/UserID/EventType; auto-sets Timestamp if zero; calls GDPR check → IP mask → PublishRaw |

**Required fields** in POST body:
```json
{ "event_id": "...", "user_id": "...", "event_type": "...", "gdpr_consent": true, "payload": {} }
```
**GDPR-gated event types**: `click`, `view_item`  
**Response**: `202 Accepted` → `{"status":"accepted","message":"event queued for screening"}`

---

### 3.2 `pkg/domain/fraud`

| File | Responsibility |
|---|---|
| `rules.go` | Three `FraudRule` implementations: BotRule, RateLimitRule, GeoVelocityRule |
| `service.go` | `FraudService.Start(ctx)` — consumer loop; `ScreenEvent()` — runs all rules, saves to DB, publishes screened |
| `repository.go` | `FraudRepository` interface: `SaveEvent`, `GetEventsByUserID`, `DeleteUserEvents` |
| `memory.go` | In-memory repository using `sync.RWMutex` maps |
| `postgres.go` | PostgreSQL repository using `database/sql` |

**Fraud Rules detail**:

| Rule | Score | Trigger condition |
|---|---|---|
| `BotRule` | 0.8 | UA contains: `bot`, `crawler`, `headless`, `spider`, `python-requests`, `curl` |
| `BotRule` (empty UA) | 0.4 | User-Agent is empty string |
| `RateLimitRule` | 0.7 | Same IP has >10 events within a 5-second sliding window |
| `GeoVelocityRule` | 0.9 | User's country changes within 10 seconds of last event |

**Fraud threshold**: `fraudScore >= 0.5` → `IsFraudulent = true`. Score is capped at `1.0`.  
**Metric keys**: `"fraudulent"`, `"legitimate"` (passed to `tracker.IncrementMetric`)

**Memory repo GDPR cross-domain delete**: Constructor `NewMemoryFraudRepository(onDelete func(userID string))` — when `DeleteUserEvents` is called, it also fires the callback, which deletes the CRM contact in Marketing's memory repo. This simulates cross-domain cascade.

---

### 3.3 `pkg/domain/marketing`

| File | Responsibility |
|---|---|
| `service.go` | Consumer loop — skips fraudulent events; routes by EventType to workflows |
| `handlers.go` | `HandleGDPRDelete` (POST), `HandleCCPAExport` (GET); SHA-256 GDPR audit log |
| `http_crm.go` | `HTTPCRMAdapter.SyncContact()` — builds POST request to CRM URL with JSON body |
| `adapter.go` | `CRMAdapter` interface definition |
| `repository.go` | `MarketingRepository` interface |
| `memory.go` | In-memory maps for CRMContacts and PrivacyLogs |
| `postgres.go` | PostgreSQL repository |
| `models.go` | `CRMContact`, `PrivacyLog` structs |

**Marketing Workflow triggers** (EventType → Workflow name):

| EventType | Workflow |
|---|---|
| `signup` | `"Welcome & Onboarding Campaign"` |
| `checkout_completed` | `"Loyalty Rewards Activation"` |
| `cart_abandoned` | `"Recovery Email Sequence"` |
| anything else | **skipped** (early return) |

**Metric keys**: `"crm_attempts"`, `"crm_successes"`, `"crm_failures"`

**GDPR Delete flow** (`POST /v1/privacy/gdpr-delete`, body: `{"user_id":"..."}`):
1. `fraudService.DeleteUserEvents(userID)` — cross-domain purge
2. `repo.DeleteCRMContact(userID)` — local CRM purge  
3. SHA-256 hash of `userID + salt` → stored in `privacy_logs` table as audit entry  
4. Salt constant: `"SentinelFlowSuperSecretSaltForGDPRCompliance"`  
5. Response: `{"status":"purged","user_id_hashed":"<sha256>","records_scrubbed":<N>}`

**CCPA Export flow** (`GET /v1/privacy/ccpa-export?user_id=...`):
1. `fraudService.GetEventsByUserID(userID)` — full event history
2. `repo.GetCRMContact(userID)` — CRM record
3. Response: `{"user_id":"...","events":[...],"crm_contact":{...},"exported_at":"..."}`

### 3.4 `pkg/domain/analytics`

| File | Responsibility |
|---|---|
| `service.go` | `HandleMetrics` → `GET /v1/metrics` (JSON); `HandleDashboard` → `GET /v1/dashboard` (text/plain formatted report) |
| `tracker.go` | `TelemetryTracker` interface: `IncrementMetric(name, delta)`, `GetMetrics()` |
| `memory.go` | `MemoryTelemetryTracker` — atomic counters (`sync/atomic`), tracks `startTime` for `EventsPerSecond` calc |
| `postgres.go` | `PostgresTelemetryTracker` — reads from `metrics_counters` table; also implements `IncrementMetric` for all domains |
| `models.go` | `Metrics` struct — `TotalEventsReceived`, `LegitimateEvents`, `FraudulentEvents`, `CRMAttempts`, `CRMSuccesses`, `CRMFailures`, `EventsPerSecond` |

**Shared Tracker pattern**: In Postgres mode, a single `PostgresTelemetryTracker` instance satisfies **all four domain tracker interfaces** (`ingest.TelemetryTracker`, `fraud.TelemetryTracker`, `marketing.TelemetryTracker`, `analytics.TelemetryTracker`). Possible because all four interfaces define the same method signature: `IncrementMetric(name string, delta int64)`.

**Metric names used**:

| Domain | Key |
|---|---|
| Ingest | `total_received` |
| Fraud | `legitimate`, `fraudulent` |
| Marketing | `crm_attempts`, `crm_successes`, `crm_failures` |

## 4. Shared Infrastructure

### 4.1 `pkg/broker`

**Interface** (`broker.go`):
```go
type Broker interface {
    PublishRaw(ctx, *TrackingEvent) error
    SubscribeRaw(ctx) (<-chan *TrackingEvent, error)
    PublishScreened(ctx, *ScreenedEvent) error
    SubscribeScreened(ctx) (<-chan *ScreenedEvent, error)
    Close() error
}
```

**Event types** (`events.go`):
```go
type TrackingEvent struct {
    EventID, UserID, EventType, IPAddress, UserAgent string
    Payload     map[string]interface{}
    Timestamp   time.Time
    GDPRConsent bool
}

type ScreenedEvent struct {
    TrackingEvent
    IsFraudulent bool
    FraudScore   float64
    FraudReason  string
    ScreenedAt   time.Time
}
```

**Redis channels**: `sentinel_flow_raw` (raw events), `sentinel_flow_screened` (screened events)  
**Redis retry vars** (test-configurable): `broker.PingRetries = 10`, `broker.PingDelay = 1s`

### 4.2 `pkg/resilience`

**CircuitBreaker states**:

```
CLOSED  ──(N failures >= threshold)──▶  OPEN
OPEN    ──(cooldown elapsed)──────────▶  HALF-OPEN
HALF-OPEN ──(success)─────────────────▶  CLOSED
HALF-OPEN ──(failure)─────────────────▶  OPEN
```

**Production config** (in `cmd/monolith/main.go`):
```go
resilience.NewResilientClient(
    250*time.Millisecond,  // HTTP timeout
    3,                     // CB failure threshold
    5*time.Second,         // CB cooldown before half-open
    2,                     // max retries
    20*time.Millisecond,   // base delay (exponential backoff)
    onCBChange,            // state transition callback
)
```

**Backoff formula**: `delay = baseDelay * 2^(attempt-1) + rand(0..100ms)`  
**CB triggers on**: connection error OR HTTP status >= 500  
**MockCRM behavior**: 5% timeout (3s hang), 15% 503, 80% success

### 4.3 `pkg/mockdb` — Custom SQL Driver

**Error injection flags** (set in test, reset after):

| Variable | Type | Effect |

## 7. Testing Strategy — 100.0% Statement Coverage

### Rule 1: Exit Crasher Subprocess Pattern
Used for every `os.Exit(1)` path in main functions:

```go
func TestMainExitDB(t *testing.T) {
    if os.Getenv("BE_CRASHER_DB") == "1" {
        os.Setenv("DATABASE_TYPE", "postgres")
        os.Setenv("DATABASE_URL", "postgres://invalid:invalid@localhost:9999/invalid")
        main()
        return
    }
    cmd := exec.Command(os.Args[0], "-test.run=TestMainExitDB")
    cmd.Env = append(os.Environ(), "BE_CRASHER_DB=1")
    err := cmd.Run()
    if e, ok := err.(*exec.ExitError); ok && !e.Success() {
        return // ✅ verified exit code != 0
    }
    t.Fatalf("expected non-zero exit, got: %v", err)
}
```

### Rule 2: mockdb Error Injection Pattern
Always reset flags after test to prevent leakage:

```go
func TestSomeDBError(t *testing.T) {
    mockdb.SimError = errors.New("forced error")
    defer func() { mockdb.SimError = nil }()
    // ... call function under test
}
```

### Rule 3: Redis Retry Speed-up
Reduce retries in tests to avoid 10s waits:

```go
broker.PingRetries = 1
broker.PingDelay = 1 * time.Millisecond
```

## 8. Strict Code Constraints

1. **Max 200 lines per `.go` file**. If a file grows, split into `_errors_test.go`, `helpers.go`, etc.
2. **No external web frameworks**. Use `net/http` + `http.ServeMux` only.
3. **No global state leakage** between tests. Always `defer` reset of `mockdb.*` flags.
4. **Goroutines must respect `ctx.Done()`**. All consumer loops use `select` with `case <-ctx.Done()`.
5. **Structured logging only** via `log/slog`. No `fmt.Println` in production code.
6. **Crash exits (`os.Exit`) only in `cmd/` packages**. Never in `pkg/` library code.
